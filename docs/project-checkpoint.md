# Checkpoint formal del proyecto final - Clase 08

## Que es este checkpoint

Este checkpoint **no es la entrega final**.

Tampoco es una defensa completa.

Es una validacion de direccion.

Busca confirmar que:

- entendiste el problema
- acotaste un alcance razonable
- tienes una primera propuesta defendible
- puedes explicar decisiones iniciales de base de datos

## Que debes traer

Debes poder mostrar, de forma breve y clara:

- problema resumido
- alcance elegido
- entidades principales
- relaciones principales
- 5 reglas de negocio
- diagrama o esquema logico inicial
- 2 consultas relevantes
- 1 decision tecnica importante
- 1 riesgo o duda

## Regla de presentacion

Cada presentacion es individual y tiene hasta `5 minutos`.

Distribucion sugerida:

1. caso y problema central
2. diagrama o esquema logico actual
3. propuesta inicial de solucion

## Que no se espera todavia

No se espera:

- frontend
- backend completo
- dashboard final
- despliegue
- documentacion exhaustiva
- proyecto terminado

## Como se revisara

La revision sera breve y orientada a claridad y direccion.

Formato sugerido:

1. Problema y alcance
2. Entidades y relaciones
3. Reglas y consultas
4. Decision tecnica y riesgo

## Que revisara el instructor

- si el problema se entiende
- si el alcance esta acotado
- si las entidades y relaciones tienen sentido
- si las reglas son concretas
- si las consultas representan necesidades reales
- si puedes defender una decision tecnica inicial
- si reconoces un riesgo o una duda

## Como se conecta con la entrega final

El checkpoint sirve para:

- corregir direccion a tiempo
- evitar proyectos demasiado grandes
- detectar huecos tempranos
- llegar mejor a la entrega final

No sustituye la entrega final.

No congela por completo el diseño, pero si debe dejar una direccion mas clara.
