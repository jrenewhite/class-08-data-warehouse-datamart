# Escenario

Usaremos un mini-caso de analisis de ventas y pagos.

Fuentes simples:

- pedidos
- productos
- clientes
- pagos
- sucursales

La demo lanzara dos preguntas:

1. una necesidad mas amplia:
   - comparar ingresos, clientes y productos de forma integrada
2. una necesidad mas enfocada:
   - seguir ventas por sucursal y categoria para un equipo comercial

La idea es observar cuando parece mas razonable una capa tipo warehouse y cuando parece mas razonable una capa tipo datamart.
