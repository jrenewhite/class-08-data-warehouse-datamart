# Domain Model

## Tablas del demo

### `customers`

- cliente del sistema

### `products`

- producto y categoria

### `branches`

- sucursal

### `orders`

- pedido operativo

### `payments`

- pago asociado al pedido

## Intencion del modelo

El dominio es deliberadamente pequeño.

Solo buscamos suficiente contexto para discutir:

- que integrarias en una capa analitica mas central
- que servirias en una capa mas especializada
- que tradeoff aparece
